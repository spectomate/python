Architecture

Spectomate is built on a modular architecture that makes it easy to extend and add new converters between different Python package management formats.

Key Components

  • BaseConverter - Abstract base class that defines the common interface for all converters
  • Data Schemas - Define the data structure for each format
  • Converters - Implement the logic for conversion between formats
  • Converter Registry - Manages registration and discovery of converters

Converters

Converters are responsible for transforming data between different package management formats.

Available Converters

  • CondaToPipConverter - Converts from conda format to pip format
  • PipToCondaConverter - Converts from pip format to conda format
  • PipToPoetryConverter - Converts from pip format to poetry format
  • PoetryToPipConverter - Converts from poetry format to pip format
  • PoetryToCondaConverter - Converts from poetry format to conda format
  • PipenvToPipConverter - Converts from pipenv format to pip format
  • PipenvToPoetryConverter - Converts from pipenv format to poetry format
  • SetupToPipConverter - Converts from setup.py format to pip format
  • SetupToPoetryConverter - Converts from setup.py format to poetry format

Example Usage

from spectomate.converters.pip_to_poetry import PipToPoetryConverter

converter = PipToPoetryConverter(
    source_file="requirements.txt",
    target_file="pyproject.toml",
    options={"project_name": "myproject", "version": "0.1.0"}
)
result_path = converter.execute()

Data Schemas

Data schemas define the structure for each package format and provide methods for parsing and generating files.

Available Schemas

  • PipSchema - Schema for pip format (requirements.txt)
  • CondaSchema - Schema for conda format (environment.yml)
  • PoetrySchema - Schema for poetry format (pyproject.toml)
  • PipenvSchema - Schema for pipenv format (Pipfile)
  • SetupSchema - Schema for setup.py format

API Reference

Spectomate can be used as a Python library in your projects.

Basic Usage

from spectomate.core.registry import get_converter

# Get a converter based on source and target formats
converter = get_converter("pip", "poetry")(
    source_file="requirements.txt",
    target_file="pyproject.toml",
    options={"project_name": "myproject", "version": "0.1.0"}
)

# Execute the conversion
result_path = converter.execute()

Advanced Usage

from spectomate.core.registry import get_converter
from spectomate.schemas.pip_schema import PipSchema
from spectomate.schemas.poetry_schema import PoetrySchema

# Parse source file manually
source_data = PipSchema.parse_file("requirements.txt")

# Get converter
converter_class = get_converter("pip", "poetry")
converter = converter_class(
    source_file=None,  # We'll provide data directly
    target_file="pyproject.toml",
    options={"project_name": "myproject", "version": "0.1.0"}
)

# Convert data manually
converter.source_data = source_data
target_data = converter.convert(source_data)

# Generate target file
PoetrySchema.generate_pyproject_toml(target_data, "pyproject.toml")

CLI Usage

Spectomate provides a command-line interface for easy usage.

Basic Commands

# Convert from pip to poetry
spectomate -s pip -t poetry -i requirements.txt -o pyproject.toml --project-name myproject --version 0.1.0

# Convert from conda to pip
spectomate -s conda -t pip -i environment.yml -o requirements.txt

# Convert from poetry to conda
spectomate -s poetry -t conda -i pyproject.toml -o environment.yml

# Convert from pipenv to pip
spectomate -s pipenv -t pip -i Pipfile -o requirements.txt --include-dev

# List available converters
spectomate --list-converters

About Spectomate

Spectomate is an open-source project designed to simplify the conversion between different Python package management formats. It was created to solve the common problem of migrating between package management systems.
